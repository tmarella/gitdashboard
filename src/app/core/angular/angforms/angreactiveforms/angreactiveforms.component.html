Reactive Forms

Reactive forms were introduced with Angular 2. We build our own representation of a form in the component class. The main advantange is we can not unit test the form validation logic.

Setup
  • Import ReactiveFormsModule in your app or feature module
  • Simple Reactive form template
  • {{reactiveForm}}
  • Component class
  • {{componentClass}}
  • We can see that the form is really just a FormControl, which keeps track of the global validity state.
  • In the array notation, the first element of the array is the initial value of the control, and the remaining elements are the control's validators
  • It is possible to subscribe to the Form stream of values using the Observable API
  • {{observable}}
  • We can get to a control with either myForm.get(‘name’) or with myForm.controls.name
  • We can update form values using patchValue() and setValue()
  • We can get to errors with either .hasError(‘required’) or .errors.required
  • patchValue(): We use patchValue to set the subset of the values, we then use patchValue. In this example, we set only of the three values.
    setValue(): We use setValue to set the value of every form of control on the form. For example, here we have three form control in our form model. And now because we want to update the value of each one of these, we use setValue. 
    {{updateForm}}
  • To reset the form, we use reset()
Class Description
AbstractControl The abstract base class for formControl, formGroup and formArray
FormControl Manages the value and validity status of individual form control
FormGroup Manages the value and validity status of group of form controls
FormArray Manages the value and validity status of arrary of form controls
FormBuilder An injectible service that provides factory methods for creating control instances
