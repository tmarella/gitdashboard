Angular Building Blocks

Angular is a framework built using TypeScript that provides developers with robust tools for creating the client side of web applications and it follows component based architecture.

Building Blocks
  • Modules
    • Modules are aimed for modularity and denoted by @NgModule.
    • The purpose of a NgModule is to declare each thing you create in Angular and group them together, like packages in Java or namespaces in C#.
    • The @NgModule decorator is used to later on define the imports, exports, declarations, providers and bootstrapping options.
    • The BrowserModule import is required by default for any web based angular application.
    • The bootstrap option tells Angular which Component to bootstrap in the application.
    • Export array is used to export components, directives, and pipes which can then be used in other modules.
  • Components
    • Component is a piece of the screen denoted by @Component.
    • constructor - The constructor is not actually part of Angular. It is feature of Typescript class, which is called when class is instantiated.
    • ngOnInit − This is called whenever the initialization of the directive/component after Angular first displays the data-bound properties happens.
  • Templates
    • The HTML templates have the look of how the page will look.
    • The page display can be defined internally — by template: and a piece of code, or externally — by templateUrl: and a directory to the compatible .html file.
  • Directives
    • Directives in Angular lets you change the UI elements at runtime and denoted by @directive.
    • Refer here for more information Directives and Pipes
  • Services and Dependency Injection
    • A service is typically a class with a narrow, well-defined purpose and denoted by @Injectable.
    • DI is inserting the service in the component’s constructor to make the code modular, reusable, and efficient.
    • A dependency doesn't have to be a service—it could be a function, for example, or a value.
    • We can register providers in the metadata of the service (in the @Injectable() decorator), or in the @NgModule() or @Component() metadata
  • Metadata
    • The metadata for a component tells Angular where to get the major building blocks that it needs to create and present the component and its view.
    • Few options are selector, template, styles, animations, providers
  • Angular Built-in security
    • Built in support to prevent cross-site request forgery (XSRF) vulnerabilty - The application server sends a randomly generated authentication token in a cookie. The client code reads the cookie and adds a custom request header with the token in all subsequent requests.
    • Built in support to prevent cross-site script inclusion (XSSI) vulnerabilty - Angular recognizes the convention and automatically strips the string ")]}',\n" from all responses before further parsing.
    • Preventing XSS using Sanitizaion. Angular inspects untrusted value and turns it into a value that's safe to insert into the DOM
    • Angular provides route guards that tells the router whether it should allow navigation to a requested route. The guards are CanActivate, CanActivateChild, CanDeactivate, CanLoad and Resolve
Hook Purpose
constructor The constructor is not actually part of Angular. It is feature of Typescript class, which is called when class is instantiated.
ngOnChanges This method is called whenever one or more input properties of the component changes. The hook receives a SimpleChanges object containing the previous and current values of the property.
ngOnInit This is called whenever the initialization of the directive/component after Angular first displays the data-bound properties happens. It initializes the component and sets the input properties of the component.
ngDoCheck It gets called after ngOnChanges and ngOnInit and is used to detect and act on changes that cannot be detected by Angular.
ngAfterContentInit It gets called after the first ngDoCheck hook. This hook responds after the content gets projected inside the component.
ngAfterContentChecked This is called in response after Angular checks the content projected into the component.
ngAfterViewInit This is called in response after Angular initializes the component's views and child views.
ngAfterViewChecked This is called in response after Angular checks the component's views and child views.
ngOnDestroy This is the cleanup phase just before Angular destroys the directive/component.
