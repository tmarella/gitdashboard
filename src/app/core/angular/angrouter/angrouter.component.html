Routing

Routing is an integral part of any single page application and it allows a user to navigate from one view to the next.

The @angular/router contains the shown terms:

  • Service: This is a global service present inside the Angular Application.
  • State: The state of Router is maintained by Router State.
  • Configuration: It maintains information about all the routing states which is present inside the application.
  • Guard: This is a code that runs whenever the router is activated, deactivated or loaded.
  • Snapshot: It will provide access to state and data for a particular route node.
  • Resolver: The script that will fetch data before the requested page is initiated.
  • Outler: This is the location in DOM for router outlet to place itâ€™s components.
Setup
  • Base Href: Initial point of entry for routing is added by the CLI by default in index.html
  • Import the RouterModule and Routes from @angular/router into your app.module.ts
  • pathMatch: specifies a strict matching of path to reach the destination page.
  • empty path: An empty path is our default route. I.e. when the app loads for the first time.
  • ** path: Two asterisks means that the route is a wildcard. It is particularly advantageous for error reporting
  • Pass the routes array in the RouterModule using RouterModule.forRoot(routes) and have to add it to imports
  • <router-outlet></router-outlet> in the component html will ensure that the DOM of the component should be placed here.
  • To use parameterized routes, we attach, /: after our parent route
  • Routes can also be nested within one another, forming parent/child route relationships
  • Router Links: In order to actually navigate from one route to the next, you will need to use the Angular equivalent of href
  • Angular offers a way for determining what is the current active link. routerLinkActive="active"
  • Eager loading is loading the component by default without any user involvement
  • Lazy loading is loading the component on demand(only when user activates)
