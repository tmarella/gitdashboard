Component Interaction

Component interaction is a feature where the data is shared between components. There are different ways to accomplish this.

@Input: Parent → child
  • Works using the @Input() decorator to allow data to be passed via the template
  • The data flow is uni-directional from parent to child
  • {{inputParent}}
  • {{inputChild}}
@Output: Child → Parent
  • Here we listen for changes on a certain value in the child component and emit a value to the parent when an event occurs
  • Works using the @Output() and EventEmitter()
  • {{outputParent}}
  • {{outputChild}}
@ViewChild: Child → Parent
  • ViewChild allows a one component to be injected into another, giving the parent access to its attributes and functions.
  • Works using @ViewChild
  • We need to implement the AfterViewInit() lifecycle hook to receive the data from the child.
  • {{viewchildParent}}
  • {{viewchildChild}}
Template variable: Parent → child
  • We use template variable to access child members from parent
  • {{templateVariableParent}}
  • {{templateVariableChild}}
Shared service: Two independent components
  • Shared service is used for passing data between components that lack a direct connection
  • We use RxJS BehaviorSubject
  • Prefer BehaviorSubject over Subject because
    • It will always return the current value on subscription â€” there is no need to call onnext
    • It has a getValue() function to extract the last value as raw data.
    • It ensures that the component always receives the most recent data.
  • All components receive the same treatment. We inject the DataService in the constructor, then subscribe to the currentMessage observable
  • {{sharedservice}}
  • {{sharedserviceComponent1}}
  • {{sharedserviceComponent2}}
