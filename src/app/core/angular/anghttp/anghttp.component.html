HTTP

Front end applications communicate with backend servers through REST APIs (which are based on the HTTP protocol) using either the XMLHttpRequest interface or the fetch() API

The HttpClient is available from the @angular/common/http

Advantages
  • Has many built-in features for implementing test units
  • Makes use of RxJS Observables for handling asynchronous operations
  • Easy error handling
  • Retrying failed HTTP requests
  • HttpClient has built-in XSRF protection mechanisms. It sets the X-XSRF-TOKEN in the request headers with the XSRF token and server can verify it.
Prerequisites for making HTTP request from Angular
  • Add the HttpClientModule to the imports array of your AppModule
  • Import the HttpClient into your component and add it to the constructor()

Promise
  • Promise emits a single value
  • Promise is eager as it executes immediately on creation
  • Promise cannot be cancellable
  • Doesn't provide any operators
  • {{promise}}
Observable
  • Observable emits multiple values
  • Observable is lazy as it doesn't request until it is subscribed
  • Observable have subscriptions that are cancellable using the unsubscribe() method
  • Provides the map for forEach, filter, reduce, retry, and retryWhen operators.
