Introduction

ASP.NET Web API is a framework for building HTTP services and is similar to ASP.NET MVC since it contains the MVC features such as routing, controllers, action results, filter, model binders, and dependency injection.

Features
  • Supports convention-based CRUD actions, since it works with HTTP verbs GET,POST,PUT and DELETE
  • Responses have an Accept header and HTTP status code
  • Supports multiple text formats like XML, JSON etc
  • Supports Self-hosting or IIS Hosting

  • Based on HTTP and so easy to define, expose and consume in a REST-ful way
  • Open source
HTTP Methods
  • GET: Used to read a representation of resource
  • POST: Used to create new resource
  • DELETE: Used to delete a resource
  • PUT: Used to update a resource. It can be used for create also
  • PATCH: Used to modify a resource
REST principles
  • It should be state less
  • It should access all the resources from the server using URI only
  • It does not have inbuilt encryption
  • It does not have session
  • It only uses HTTP protocol
  • It should return the result only in light weight data format like JSON, XML
  • It should use HTTP verbs like GET, PUT, POST, PATCH, DELETE
Configure Web API
  • Web API supports code based configuration. It cannot be configured in web.config file
  • Web API configuration process starts when the application starts. It calls GlobalConfiguration.Configure(WebApiConfig.Register) in the Application_Start method
  • The Configure() method requires the callback method where Web API has been configured in code. By default this is the static WebApiConfig.Register() method
  • Register() accepts a parameter HttpConfiguration that contains properties like Filters, Formatters, properties, Routes, Services
