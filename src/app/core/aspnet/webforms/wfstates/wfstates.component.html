State

Cookie, Sessions and ViewState are used to maintain value and validity of data

Cookie

Cookies are small pieces of text, stored on the client's computer to be used only by the website setting the cookies.

  • We can create a new cookie value: HttpCookie cookie = new HttpCookie("BackgroundColor");
  • We can assign the value: cookie.Value = ColorSelector.SelectedValue;
  • We can set the lifespan of cookie: cookie.Expires = DateTime.Now.AddHours(1);
  • We can set the value: Response.SetCookie(cookie);
  • When the page loads, In Page_Load event, we will find if the value of the cookie: Request.Cookies["BackgroundColor"]
  • If the cookie value is not null, we will set the value
  • Cookies are available across the application
Sessions

Sessions can be used to store even complex data for the user. Actually, sessions will use cookies to store the data, unless you explicitly tell it to store in session object.

  • We can set the value initally: Session["BackgroundColor"] = ColorSelector.SelectedValue;
  • We can get the value and assign it: ColorSelector.SelectedValue = Session["BackgroundColor"].ToString();
  • We can set the session lifespan in web.config file
  • Session values are available across the application
ViewState

ViewState allows ASP.NET to repopulate form fields on each postback to the server

  • We can set the value: ViewState["NameOfUser"] = NameField.Text;
  • We can get the value: ViewState["NameOfUser"]
  • ViewState is available only in that specific page
