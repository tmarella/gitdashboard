Exception Handling

Other than exceptions handled in Try-Catch blocks, there are some unhandled exceptions that can be handled using below four types.

  • Using <customErrors> element in web.config
  • {{customErrors}}
  • Using HandleErrorAttribute
    • HandleErrorAttribute inherits from FilterAttribute and can be applied to an entire controller or individual action methods.
    • It can only handle 500 level errors
    • The problem with HandleErrorAttribute is it doesn’t provide a way to log the exception
    • Note: HandleErrorAttribute requires customErrors to be enabled in your Web.Config.
    • {{sqlException}}
  • Overriding Controller.OnException()
    • OnException is similar to HandleErrorAttribute
    • Works with all HTTP status codes and also gives you the ability to log the errors and present custom MVC views 
    • {{onException}}
  • Using Application_Error event of HttpApplication
    • To log all unhandled exceptions that may occur within your application, you should implement basic error logging code
    • {{applicationError}}
    • When to use: Always! HttpApplication’s Error provides the best mechanism to collect and log all unhandled application errors.
