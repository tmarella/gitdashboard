Routing

Routing is the process of directing an HTTP request to a controller

Convention based Routing
  • Url Pattern: {domain}/{controller}/{action}/{id}
  • We can configure a custom route using MapRoute extension method. You need to provide at least two parameters in MapRoute, route name and url pattern. The Defaults parameter is optional.
  • After configuring all the routes in RouteConfig class, you need to register it in the Application_Start() event in the Global.asax
    RouteConfig.RegisterRoutes(RouteTable.Routes);
  • MVC framework evaluates each route in sequence.
  • Simple Convention Routing example
    • {{conventionRouting}}
  • Convention Routing example with constraints
    • We can apply restrictions on the value of parameter by configuring route constraints
    • {{conventionRoutingConstraints}}
  • We can register multiple custom routes with different names
    • {{conventionRoutingMultipleConstraints}}
Attribute based Routing

Attribute Routing (introduced in MVC 5) is the ability to add routes to the Route Table via attributes so that the route definitions are in close proximity to their corresponding actions.

  • To enable attribute routing, routes.MapMvcAttributeRoutes();
