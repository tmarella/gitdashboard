Filters

Filters are custom classes that provide logic to execute pre-action and post-action of controller action methods.

Four Types
  • Authorization filters: Performs authentication and authorizes before executing action method like [Authorize]
  • Action filters: Performs some operation before and after an action method executes
    • Action filters are attributes that can be applied either on a controller or controller action method.
    • When applied on a controller level, it is applicable for all actions with in that controller.
  • Result filters: Performs some operation before or after the execution of view result like [OutputCache]
  • Exception filters: Performs some operation if there is an unhandled exception like [HandleError]
Register Filters
  • Global level: Applies at global level in the Application_Start event of Global.asax.cs file by using default FilterConfig.RegisterGlobalFilters() method
  • Controller level: Applies to the controller class. So, filters will be applicable to all the action methods
  • Action method level: Applies filters to an individual action method
Custom Filter
  • We can create custom filter attributes by implementing an appropriate filter interface
  • For example, implement IExceptionFilter and FilterAttribute class to create custom exception filter.
  • implement an IAuthorizatinFilter interface and FilterAttribute class to create a custom authorization filter.
