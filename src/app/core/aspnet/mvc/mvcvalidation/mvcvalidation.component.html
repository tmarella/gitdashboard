Validation

Validation is used to check whether the user input is valid. Validations in MVC can be performed using:

    Client Side Validation

    • Jquery
      • Enable ClientValidation and UnobtrusiveJavaScript in Web.config file
      • {{jQuery}}
      • Include jQuery.min, jQuery.validate, jQuery.unobtrusive scripts in the view.

    Server Side Validation

    • ModelState Object
      • ModelState is a property of controller that is used for server side validation (System.Web.Mvc).
      • ModelState is a collection of name and value pair submitted to the server during post
      • ModelState takes 2 parameter. First Parameter “Name” is a name of input control and 2nd parameter is a string error message.
      • We need to check boolean property "IsValid" in ModelState object for validation.
        • {{modelStateHtml}}
      • On controller we add error message to ModelState and set to the control like ModelState.AddModelError("Name", "Name Required");.
        • {{modelStateCs}}
    • Data Annotation
      • Has built-in validation attributes for different validation rules, applied to the properties of model class
      • Ther are included in System.ComponentModel.DataAnnotations
      • In the Attribute we can add error message
      • Required, RegularExpression, EmailAddress, Phone, MaxLength, MinLength etc
    • Custom validations
      • For creating "Custom Validation" we need to create one class which is derived from ValidationAttribute class.
      • we have to override the "IsValid" function and all the custom validation logic should go inside it.
      • The requirement here is, Join date of customer should be less than current date
      • {{customValidator}}
