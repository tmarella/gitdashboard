Architecture

MVC stands for Model, View, and Controller. MVC separates an application into three components - Model, View, and Controller.

  • Model represents the shape of the data. A class in C# is used to describe a model. Model objects store data retrieved from the database.
  • View is a user interface. View displays model data to the user and also enables them to modify them.
  • Controller handles the user request. Typically, the user uses the view and raises an HTTP request, which will be handled by the controller

Folder structure

  • App_Data: Contains application data files like LocalDB, .mdf files, xml files
  • App_Start: Contains class files(AuthConfig.cs, BundleConfig.cs, FilterConfig.cs, RouteConfig.cs) which will be executed when the application starts
  • Content: Contains static files like css files, images and icons files
  • Controllers: Contains class files for the controllers
  • Fonts: Contains custom font files
  • Models: Contains model class files that includes public properties
  • Scripts: Contains JavaScript or VBScript files
  • Views: Contains .cshtml files
  • Global.asax: Contains Code related to application level events(Application_BeginRequest, application_start, application_error, session_start, session_end)
  • Packages.config: File is managed by NuGet to keep track of what packages and versions installed in application
  • Web.config: Contains application level configurations.
