Model Binding

Model binding maps data from HTTP requests to action method parameters

An HTTP request can carry data in different ways

  • Request.Form (values submitted in a form as an HTTP post request)
  • RouteData.Values (route value,s such as /Person/Edit/6)
  • Request.QueryString (data passed in from the query string part of a URL, such as /Person/Edit?id=6)
  • Request.Files (uploaded files)
HTTP Get
  • The model binder will match parameter names in the method signature to those found in the request.
  • {{httpGet}}
  • if the parameter is not provided, then the model binder will bind a null value for that parameter, if the parameter is a nullable type.
HTTP Post
  • The model binder will bind the whole model directly to the action method
  • {{httpPost}}
Bind attribute
  • Bind attribute is used to whitelist or blacklist specific properties
  • {{bindAttribute}}
  • Using the Bind attribute is one way to reuse the same model class while only caring about a subset of its properties
Custom model binder
  • When/Why to use: Sometimes, the property of the model does not match with the UI page.
  • Need to implement IModelBinder
  • {{customBinder}}
  • Need to register our custom model binder either in Application_Start() or directly in the action method
  • {{applicationStart}}
  • {{attributeControllerAction}}
