Caching

Caching means to store something in memory that is being used frequently to provide better performance.

Two Ways
  • Page Output Caching: Refers to the ability of the web server to cache a certain webpage
    • Output Cache Filter
      • Allows you to cache the data that is output of an action method
      • By default, this attribute filter cache the data till 60 seconds
      • To enable Output Caching, add [OutputCache] attribute to an action method
      • If we have multiple action methods across controllers needing the same caching behavior, we can put the caching values in the web.config and create a cacheprofile for it. 
      • {{cacheProfileConfig}}
      • Usage of cache profile is
      • {{cacheProfileUsage}}
      • Parameters used: Duration, SqlDependency, Location
      • Output Caching Location
        • By default, content is cached in three locations: the web server, any proxy servers, and the user's browser
        • Content's cached location can be set to: Any, Client, Downstream, Server, None, or ServerAndClient.
  • Application Caching: Application data caching is a mechanism for storing the Data objects in a Key-Value based cache.
    • {{applicationCaching}}
    • Parameters used: absoluteExpiration, Dependencies, onRemoveCallBack
Key points
  • Avoid caching for contents that are unique per user.
  • Use caching for contents that are accessed frequently.
  • Use the VaryByCustom function to cache multiple versions of a page based on cookies, role, theme, browser, and so on.
Advantages
  • Reduces server requests
  • Reduces database requests
  • Reduces network traffic
  • Avoids time-consumption for regenerating reusable content
  • Improves performance
