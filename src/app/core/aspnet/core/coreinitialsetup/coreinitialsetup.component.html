Introduction

  • Program.cs file contains Main()
  • Main() is the place where the configuration takes place. CreateWebHostBilder() returns IWebHostBuilder and does below tasks
    • Setting up the web server
    • Loading the host and application configuration
    • Configuring logging
  • Startup.cs file is the entry point of the ASP.NET Core application and it contains ConfigureServices() and Configure()
    • ConfigureServices(): configures the servies required for the application
    • Configure(): configures application request processing pipeline
Features
  • Built-in supports for Dependency Injection
  • Built-in supports for the logging framework and it can be extensible
  • Introduced new, fast and cross-platform web server - Kestrel. So, a web application can run without IIS, Apache, and Nginx
  • Multiple hosting ways are supported
  • Command-line supports to create, build and run the application
  • There is no web.config file. We can store the custom configuration into an appsettings.json file
  • There is no Global.asax file. We can now register and use the services into startup class
  • Provide protection against CSRF (Cross-Site Request Forgery)
Advantages of ASP.NET Core over ASP.NET
  • It is cross-platform, so it can be run on Windows, Linux, and Mac.
  • There is no dependency on framework installation because all the required dependencies are ship with our application
  • ASP.NET Core can handle more requests than the ASP.NET
  • Multiple deployment options available withASP.NET Core
Hosting
  • InProcess hosting: host the app inside the IIS(iisexpress.exe) and this is efficient.
  • OutofProcess hosting
    • Internal web server: kestrel
    • External web server: can be IIS, Nginx or Apache
Settings
  • launchSettings.json: This file is only used on local development machine. No need to publish. It can define project specific settings associated with each profile Visual Studio is configured to launch the application, including any environment variables that should be used. You can define framework for your project for compliation and debugging for specific profiles.
  • appSettings.json
    • Stores application configuration settings like DB connection strings,Logging in this file.
    • We can have environment specific appsettings.json files
Configuration Sources
  • Files(appSettings.json)
  • User Secrets
  • Environment variable
  • Command line arguments

If you have a configuration setting with the same key in multiple configuration sources, the later configuration sources override the earlier configuration sources

  • appSettings.json
  • appSettings.{environment}.json
  • User Secrets
  • Environment variable
  • Command line arguments
