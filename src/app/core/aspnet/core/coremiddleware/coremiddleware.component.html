Core Middleware

Middleware is a piece of software that can handle an HTTP request or response.

  • middleware component(MC) has access to both - the incoming request and the outgoing response
  • MC may process the incoming request and pass it to next middleware in pipeline
  • If the request is not passed to the next middleware in the pipeline. This is called short-circuiting
  • MC are executed in the order they are added to the pipeline.
  • MC are available as NuGet packages.
Configuring the request processing pipeline
  • lets say we have three MC.
  • To call next MC in pipeline, we add 2nd parameter i.e 'next' and call that using next()
  • The order of the execution is
    • MC1 Incoming request
    • MC2 Incoming request
    • MC3 Incoming request
    • MC3 Outgoing response
    • MC2 Outgoing response
    • MC1 Outgoing response
Some middleware components
UseStaticFiles Adds support for serving static files
UseDefaultFiles Adds support for serving static files outisde wwwroot folder
UseDirectoryBrowser Adds support to allow users to see files within a specified directory
UseFileServer Adds support to combine UseStaticFiles, UseDefaultFiles, UseDirectoryBrowser
UseExceptionHandler Adds support to catch exceptions, log them and re-execute in an alternate pipeline.
UseHsts Adds strict-transport-security-header
UseRouting Adds routing capabilities for MVC or web form
UseHttpsRedirection Redirects Http requests to Https
UseAuthentication Attempt to authenticate user before they are allowed to access secured resources.
UseEndpoints Adds support to add razor pages endpoints to request pipeline
UseAuthorization Authorize a user to access secured resources.
