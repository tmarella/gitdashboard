CSS

Basics
  • To add styles for a element inside an element, use element1 element2 { styles }
  • To add styles to more than one selector, use element1, element2 { styles }
  • To add styles for an element with specific class, use element1.element2 { styles }
  • To add styles using a class, use .
  • To add styles using a id, use #. Same id cannot be used twice in a html file
  • To add only styles for first element in a list, use li:first-child{ styles }
  • To add only styles for last element in a list, use li:last-child{ styles }
  • To add styles for an element when active, use element:active{ styles }
  • To add styles for an element on hover, use element:hover{ styles }
  • To add styles for an element on focus, use element:focus{ styles }
Browser dev tool features
  • Copy all the changes from the changes tab in devtools
  • Edit the animations in the animations tab
  • Check fonts from any website using fonts tab
CSS Units
  • px(Pixel) unit is going to be absolute width which is always going to be same no matter where it is defined
  • %(Percentage) unit is going to be percentage width based of it parent's size
  • vw(view width) unit is going to be percent width of the entire screen size
  • vh(view height) unit is going to be percent height of the entire screen size
  • viewport units(vw, vh) are based on entire screen size whereas % unit is relative to its parent
  • rem units are relative but relative to the font-size that is defined. r stands for root. Hence rem units are relative to the root font size
  • em units are relative to its parent.
CSS Calc
  • CSS calc is one of the few functions available in CSS and it is incredibly powerful.
  • We can perform simple arithmetic operations with the calc function, but we can also combine units together with the calc function to create really unique spacing and sizes
  • CSS calc function has the ability to use it with variables(syntax: --variable)
CSS Pseudo Elements
  • ::before and ::after are pseudo elements that allow us to add elements into the code strictly from css
  • Since CSS3, two colons(::) are required for specifying pseudo elements
  • We cannot set pseudo elements for the elements that do not have content like input elements
CSS Display
  • Default display property for div elements is 'block'. Hence it will take full width
  • Default display property for span elements is 'inline'. Hence it will take minimum amount of space possible. Inline elements cannot have height and width
  • For images, we can use 'inline-block'. So it can use minimum amount of space and also we can specify width and height
  • display: none property completely deletes the element from the html
CSS Media Queries
  • Media queries are used to change styles based on size of device(mobiles)
  • Media query starts with @media and followed by type of device(screen, print, speech)
  • By default, we can select all devices. The syntax is @media all followed by paranthesis
  • Even if we don't use 'all', it defaults to 'all'.
  • Media queries should be written at the end of the styles file. So that the styles dont get overridden.
  • Media queries can be used to style an element based on orientation(landscape, potrait)
  • Styles can be applied based on one or more conditions by using and keyword. syntax is @media (orientation: landscape) and (max-width: 500px)
  • We can use 'or' keyword using 'comma' between multiple checks
CSS Specificity
  • Inline styles are more specific than ids, ids are more specific than classes, classes are more specific than tags
  • Example: <h1 id="header-id" class="header" style="color: red;">Header</h1>
  • Inline styles will get more priority and overrides any styles defined using id, and styles defined using id overrides any styles defined using class and styles defined using class overrides any styles defined using tags(h1 in above example)
  • Order of specificity is inline_styles>ids>class>tags
  • Fifth level of specificity can be acheived using 'important' keyword. Any styles that use 'important' keyword will override styles defined using ids, class, tags and inlinestyles
  • One way to check the specificity is using browser dev tools. If any styles you see are striked off, then this means the styles are getting overridden
CSS Variables
  • Varibles can be declared in the root element
  • Syntax of declaring a variable is --background-color: red
  • We can use the variable in any child elements using the syntax background-color: var(--background-color);
CSS Positioning
  • By default, all elements are positioned as 'static'
  • If the position is changed to relative, then we can use top, bottom, left, right to change the position of that particular element relative to the parent element
  • If the position is changed to absolute, the element position is fixed and it is now relative to the page.
  • If we want to position a child element as absolute, then we can acheive it by changing the parent position to relative
  • If the position is changed to fixed, then the element is positioned based on entire page and even if you scroll, the element stays fixed.
  • The difference between absolute and fixed is when scrolling, the element with position absolute also gets scrolled, but the element with position fixed will not get scrolled and stays where it is.
  • If the element is position as sticky, combination of relative and fixed. By default the element uses relative, as soon as we scroll, it is changed to fixed and stays at the position.
