Multi threading

Multi-threading is basically trying to do more than one thing at a time within a process.

Threads are lightweight processes and is defined as the execution path of a program.

Thread Life Cycle
  • Unstarted state: Situation when the instance of the thread is created but the Start method is not called
  • Ready state: Situation when the thread is ready to run and waiting CPU cycle
  • Not runnable state: When sleep() or wait() is called or blocked by I/O operations
  • Dead state: Situation when the thread completes execution or is aborted.
Properties
  • IsAlive: Gets a value indicating the execution status of the current thread.
  • Name: Gets or sets the name of the thread.
  • CurrentThread: Gets the currently running thread.
Methods
  • Abort(): Calling this method usually terminates the thread.
  • Sleep(): Makes the thread pause for a period of time.
  • Interrupt(): Interrupts a thread that is in the WaitSleepJoin thread state.
  • Start(): Starts a thread.
