Data Structures and Collections

Collection classes are used for data manipulation

Generic Collection
  Dictionary<TKey, TValue>
  • Represents a collection of keys and values
  • Methods: Add(TKey, TValue), Clear(), Remove(TKey)
      • Usage:
    • when we need to do fast searches and retrieval of information
    • when we need to access each element by index/key
    • when each element contains one key and one value, one key and multiple values
      List<T>
  • Represents a strongly typed list of objects that can be accessed by index
  • Methods: Add(), Clear(), Contains()
      • Usage:
    • when we need to access each element by index
    • when each element contains one key and one value, one key and multiple values
      Queue<T>
  • Represents a first-in, first-out collection of objects (FIFO)
  • Methods: Enqueue(T), Clear(), Contains(T)
      • Usage:
    • when we need a sequential list where the element is removed after its value is retrieved
    • when we need to access elements in certain order
      Stack<T>
  • Represents a variable size last-in-first-out (LIFO)
  • Methods: Push(), Pop(), Clear()
      • Usage:
    • when we need a sequential list where the element is removed after its value is retrieved
    • when we need to access elements in certain order
      LinkedList<T>
  • Represents a doubly linked list(contains previous pointer, next pointer and data)
  • Methods: AddFirst(), AddLast(), Clear()
      • Usage:
    • when we need to access elements in certain order
      ObservableCollection<T>
  • Represents a dynamic data collection that provides notifications when items get added, removed, or when the whole list is refreshed.
  • Methods: Contains(), Move(Int32, Int32), Insert(Int32, T)
    		•
      SortedList<TKey, TValue>
  • Represents a collection of key/value pairs that are sorted by key based
  • Methods: Add(TKey, TValue), Clear(), TryGetValue(TKey, TValue)
      • Usage:
    • when we need to sort the elements differently from how we entered
    • when we need to access each element by index
    Non-generic Collection
      Hashtable
  • Represents a collection of key, value pairs that are based on hash code of the key
  • Methods: Add(Object, Object), Clear(), Clone()
      • Usage:
    • when we need to access each element by index
      Array
  • Represents a strongly typed list of values that can be accessed by index
  • Methods: Reverse(Array), Clone()
    •   ArrayList
  • Represents a strongly typed list of values that can be accessed by index whose size is dynamically increased as required
  • Methods: Clone(), Clear(), Insert(Int32, Object)
      • Usage:
    • when we need to sort the elements differently from how we entered
    • when we need to access each element by index
      Queue
  • Represents a first-in, first-out collection of objects (FIFO)
  • Methods: Peek(), Clone(), Clear()
      • Usage:
    • when we need a sequential list where the element is removed after its value is retrieved
    • when we need to access elements in certain order
      Stack
  • Represents a simple last-in-first-out (LIFO) non-generic collection of objects.
  • Methods: Peek(), Pop(), Push(Object)
      • Usage:
    • when we need a sequential list where the element is removed after its value is retrieved
    • when we need to access elements in certain order
      SortedList
  • Represents a collection of key/value pairs that are sorted by the keys and are accessible by key and by index.
  • Methods: Add(Object, Object), Clone(), Clear()
      • Usage:
    • when we need to sort the elements differently from how we entered
    • when we need to access each element by index
    Collection Interfaces
      IEnumerable Enumerates(iterate) over a collection using foreach statement
      ICollection Implemented by all collections to provide copyTo(), Add() etc
      IList Used by array-indexable collections
      IDictionary Used for key/value based collections like dictionary, hastable
      IComparer Compare two objects held in a collection so that the collection can be sorted
