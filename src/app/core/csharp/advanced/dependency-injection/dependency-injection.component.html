Dependency Injection

Dependency Injection is a design pattern that allows the creation of dependant objects outside of class and provide those objects to a class in different ways.

Three Ways
  • Constructor Injection: CI is done by supplying the dependency through constructor while creating the instance of that class.
    • {{constructorInjection}}
  • Property Injection: PI is used in cases where we need a parameter-less constructor(like Struct).
    • {{propertyInjection}}
  • Method Injection: MI is used when the entire class does not need the dependency, just the one method.
    • {{methodInjection}}
Advantages
  • Reduces class coupling.
  • Code is clean and more readable.
  • Improves code maintainability.
