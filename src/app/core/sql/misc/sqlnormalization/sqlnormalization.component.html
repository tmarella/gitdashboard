Normalization

Normalization is a way of organizing the data in the database hence reducing the redundancy of data in the table and also improving the data integrity.

Why Normalization? To fix below issues
  • Insertion anomaly: It occurs when we cannot insert data to the table without the presence of another attribute
  • Update anomaly: It is a data inconsistency that results from data redundancy and a partial update of data.
  • Delete anomaly: It occurs when certain attributes are lost because of the deletion of other attributes.
Forms
  • 1NF: This form tackles the problem of atomicity. Here atomicity means values in the table should not be further divided. A single cell cannot hold multiple values
  • 2NF: The first condition in the 2nd NF is that the table has to be in 1st NF. To bring the table to Second Normal Form, we need to break the table into two parts
  • 3NF: The first condition in the 3rd NF is that the table has to be in 2nd NF. The other condition is there should be no transitive dependency. Transitive dependency is a functional dependency( X → Y and Y → Z, then X → Z).
