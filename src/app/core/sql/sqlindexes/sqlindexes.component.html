Indexes

Indexes are used to speed up the query process.

An index helps to speed up select queries and where clauses, but it slows down data input, with the update and the insert statements.

  • {{indexSyntax}}
  • Indexes should be created
    • If a column contains a wide range of values
    • If a column does not contain a large number of null values
    • One or more columns are frequently used together in a where clause or a join condition
  • Indexes should be avoided
    • The table is small
    • The columns are not often used as a condition in the query
    • The column is updated frequently
    • Two Types
    • Clustered index: Clustered index is the type of indexing that established a physical sorting order of rows. It is similar to any word Dictionary where there will be no index but sorted in alphabetical order.
    • Non-lustered index: The Non-Clustered index is an index structure separate from the data stored in a table that reorders one or more selected columns. It is similar to a textbook, the index page is created separately at the beginning of that book.
