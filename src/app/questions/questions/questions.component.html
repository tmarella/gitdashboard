When you create tables in SQL Server what are the main things that need to be taken care of?
  • Make sure you use date columns for dates, integer type columns for whole numbers that might have math done to them, VARCHAR when data width will vary, and NVARCHAR if you need to store more than one language.
  • Make sure to use third normal form--basically, avoid repetition of columns in the table
  • Make sure all your tables have a primary key and foreign key(optionsal)
  • Creating indexes on all primary keys
  • Apply Constraints on column
Explain about Breaking changes and Non breaking changes? How do you achieve it?
  • Breaking changes
    • Addition of a new required input parameter(in SP)
    • Removal of an existing input parameter (whether required or optional)
    • Deprecation of a service(ActivePDF tool for encryptpdf -> encryptfile)
    • Changing the name or data type of an existing input parameter or output value
  • Non Breaking changes
    • Addition of a new optional parameter at the end of the list of input parameters
    • Addition of new fields in the api response
    • A change in behavior to fix a bug in the service
    • Addition of a new endpoint in the existing web api
